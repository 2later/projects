Website Blog

Got so much to talk about, have to make a blog about all my awesome websites.

This website uses the github API to gather a JSON list of all my projects in my repositry. It then compiles the HTML files into a list of objects. The title is either the name of the HTML file, or the name of the parent folder. Due to my dodgy programming, there are about 100 duplicates of every website (think this is a asyncrous fetch problem).

The list of websites contains a post div which contains the title and an iFrame - this is so you get a preview of the website. It is a cool concept, but in reality is janky and bad for peformance :( Also users which use Tab to operate are gonna struggle.
